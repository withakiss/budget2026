## 🧮 분석 알고리즘

### 유사도 계산 (6종)

| 알고리즘 | 수식 | 특징 |
|---|---|---|
| **Jaccard** | `\|A∩B\| / \|A∪B\|` | 집합 기반, 단어 유무 판단 |
| **Cosine** | `A·B / (‖A‖×‖B‖)` | 벡터 각도, 빈도 반영 |
| **Dice** | `2\|A∩B\| / (\|A\|+\|B\|)` | 작은 집합에 유리 |
| **Overlap** | `\|A∩B\| / min(\|A\|,\|B\|)` | 포함 관계 탐지 |
| **TF-IDF Cosine** | `tf × log(N/df)` → cosine | 희귀 단어 가중치 |
| **🆕 BM25** | `IDF × tf(k₁+1)/(tf+k₁(...))` | 문서 길이 정규화, 포화 처리 |

### 🆕 다차원 가중합 (7차원)

```
최종 점수 = Σ(차원별 유사도 × 프로필별 가중치)

차원: TF-IDF · BM25 · Jaccard · 기능분야 · 예산규모 · 부처교차 · 법적근거
```

**분석 프로필 (6종)**

| 프로필 | 특화 분야 | 주요 가중치 |
|---|---|---|
| 기본 (default) | 전체 균형 | 텍스트 30%, 기능 20% |
| R&D 특화 | 연구개발 사업 | BM25·예산 가중치 높음 |
| 복지 특화 | 사회복지·보건 | 기능분류 가중치 높음 |
| 인프라 특화 | SOC·교통 | 예산규모 가중치 높음 |
| 디지털 특화 | IT·정보화 | 텍스트·구조 가중치 높음 |
| 인력양성 특화 | 교육·훈련 | Jaccard·기능 가중치 높음 |

### 유사도 등급 분류 (5단계)

| 등급 | 범위 | 의미 |
|---|---|---|
| 1등급 | 85% 이상 | 완전중복 |
| 2등급 | 70~85% | 고유사 |
| 3등급 | 50~70% | 부분중복 |
| 4등급 | 30~50% | 약유사 |
| 5등급 | 30% 미만 | 비유사 |

### 🆕 Value Chain 협업 패턴 탐지 (6종)

- R&D → 실증 → 사업화
- 인프라 구축 → 서비스 운영
- 인력양성 → 산업체 활용
- 데이터 구축 → AI·분석 활용
- 기반기술 → 도메인 적용
- 정책·제도 → 사업화 지원

### 🆕 낭비 리스크 스코어링

다음 5가지 요인을 합산하여 0~100점 리스크 점수 산출:

1. 급격한 예산 변동 (±80% 이상) → +25점
2. 요구액 대비 30% 이상 삭감 → +20점
3. 내역사업 예산 편중 (70% 이상) → +15점
4. 동일 기능·유형 사업 3개 이상 → +15점
5. 신규 대규모 사업 (500억 이상) → +20점

## 🔄 실제 데이터로 교체하기

`data/projects.json`의 구조를 유지하면서 내용만 교체하면 모든 차트·분석이 자동으로 업데이트됩니다.

### 데이터 소스 (추천 순서)

| 소스 | 주소 | 비고 |
|---|---|---|
| 열린재정 | [openfiscaldata.go.kr](https://www.openfiscaldata.go.kr) | 세출예산현황 Excel |
| 공공데이터포털 | [data.go.kr](https://www.data.go.kr) | "세부사업별 예산" 검색 |
| 기획재정부 | [moef.go.kr](https://www.moef.go.kr) | 예산안 첨부파일 |

### JSON 데이터 필드 설명

```json
{
  "metadata": {
    "title": "2026년도 대한민국 재정사업 현황",
    "source": "기획재정부 2026년도 예산안",
    "note": "데이터 출처 및 주의사항"
  },
  "projects": [
    {
      "id": "P001",                      // 고유 ID (필수)
      "ministry": "과학기술정보통신부",    // 부처명 (필수)
      "function_code": "140",             // 기능분류 코드
      "function": "과학기술",             // 기능분류명 (필수)
      "program": "AI·데이터",             // 프로그램명
      "project_name": "인공지능 개발",    // 사업명 (필수)
      "project_type": "R&D",             // R&D / 정보화 / 일반 (필수)
      "is_new": false,                    // 신규 여부 (필수)
      "budget_2024": 95000,              // 2024 예산 (백만원)
      "budget_2025": 115000,             // 2025 예산 (백만원, 필수)
      "budget_2026": 145000,             // 2026 예산 (백만원, 필수)
      "budget_requested": 160000,        // 요구액 (백만원, 선택)
      "change_amount": 30000,            // 증감액 (필수)
      "change_rate": 26.09,              // 증감률 % (필수)
      "account_type": "일반회계",         // 회계 유형 (필수)
      "legal_basis": "인공지능법",        // 법적 근거 (선택, 중복 탐지에 활용)
      "description": "사업 설명",         // 사업 설명 (선택, 유사도 분석 정확도 향상)
      "sub_projects": [                  // 내역사업 (선택)
        { "name": "세부사업명", "budget": 60000 }
      ]
    }
  ]
}
```

> 💡 `description`과 `legal_basis` 필드를 채울수록 유사도 분석 정확도가 높아집니다.

## 🎬 데모 시나리오

데모 때는 아래 흐름으로 보여주면 좋습니다.

1. **개요 탭**에서 총 예산, 신규사업 수, 증감 Top 10을 먼저 보여줍니다.  
2. **사업 목록 탭**에서 부처/유형/회계 필터와 CSV 다운로드를 시연합니다.  
3. **키워드 검색 탭**에서 `보안`, `클라우드`, `AI`, `복지 데이터` 같은 검색어로 TF-IDF / BM25 / Hybrid 차이를 보여줍니다.  
4. **전수스캔 탭**에서 50~70% 임계값으로 유사사업 쌍을 찾고, 부처 간 중복/협업 후보를 설명합니다.  
5. **클러스터 탭**에서 자동 군집화된 사업 묶음을 보여주며 “같은 정책군/유사사업군이 어떻게 모이는지” 설명합니다.  
6. **Value Chain 탭**에서 `R&D → 실증 → 사업화`, `데이터 구축 → AI·분석 활용` 흐름을 예시로 보여줍니다.  
7. **리스크 탭**에서 급격한 예산 변동, 요구액 대비 삭감, 신규 대규모 사업을 중심으로 이상 징후를 설명합니다.

### 추천 데모 검색어

- `보안 취약점`
- `클라우드 전환`
- `디지털 인재`
- `복지 데이터`
- `AI 실증`
- `개인정보 보호`

### 현재 샘플데이터 구성

현재 저장소의 샘플데이터에는 **전수스캔·클러스터링이 실제로 잡히도록 설계된 합성 정책사업군**이 포함돼 있습니다.  
따라서 데모 환경에서도 유사사업, 군집, Value Chain 패턴이 비어 보이지 않도록 구성돼 있습니다.
